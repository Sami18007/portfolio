1# AWS audio analyzer

Create AWS cloud resources like S3 storage, DynamoDB database table, SQS messaging service and Lambda function. Audio files can then be sent for analyzing. Sent files will be saved in the S3 bucket which triggers sending a message for the SQS service. SQS forwards the message into the Lambda function which in turn will fetch those freshly sent audio files from the S3 storage. Then, the audio files are transcribed into text and a sentiment analysis is run. The final scores are saved in the DynamoDB table from where they can be fetched to view them. Created AWS resources can be deleted through this app. All of this can be done in the console. During this project I learned a lot about Python programming and AWS cloud service.

Python, boto3, AWS cloud service
